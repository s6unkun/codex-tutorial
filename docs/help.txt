Codex CLI ヘルプ（日本語訳）

概要

Codex CLI は、インタラクティブまたは非インタラクティブな方法でモデルとやり取りするツールです。サブコマンドを指定しない場合、CLI はインタラクティブモードになります。

使い方

codex [オプション] [プロンプト]
codex [オプション] [プロンプト] <コマンド>

サブコマンド

exec：非インタラクティブにCodexを実行（別名: e）

login：ログイン管理

logout：保存された認証情報を削除

mcp：実験的：CodexをMCPサーバーとして実行

proto：標準入出力でプロトコルストリームを実行（別名: p）

completion：シェル補完スクリプトを生成

debug：内部デバッグ用コマンド

apply：Codexエージェントが生成した差分を git apply としてローカルに適用（別名: a）

help：このヘルプまたは指定したサブコマンドのヘルプを表示

引数

[PROMPT]：オプション。最初に与えるプロンプト文字列

オプション

-c, --config <キー=値>：設定ファイル（~/.codex/config.toml）の値を一時的に上書き

例: -c model="o3"

ネストされた設定にはドット区切りのパスを使う

-i, --image <ファイル>：初期プロンプトに添付する画像ファイル（複数可）

-m, --model <モデル>：使用するモデル名を指定

--oss：ローカルのOllamaサーバーを使用する設定（-c model_provider=oss 相当）

-p, --profile <プロファイル名>：config.toml内の設定プロファイルを使用

-s, --sandbox <モード>：モデルが生成したコマンドを実行する際のサンドボックスモードを選択

read-only：読み取り専用

workspace-write：作業ディレクトリ内は書き込み可能

danger-full-access：制限なし（危険）

-a, --ask-for-approval <ポリシー>：コマンド実行時に承認を求める条件を指定

untrusted：信頼されたコマンドのみ自動実行

on-failure：失敗時のみ確認

on-request：モデルが確認の要否を判断

never：常に確認なしで実行

--full-auto：--ask-for-approval on-failure + --sandbox workspace-write をまとめて指定する簡易オプション

--dangerously-bypass-approvals-and-sandbox：すべての確認・サンドボックスを無効化（非常に危険）

-C, --cd <ディレクトリ>：作業ディレクトリを指定

-h, --help：このヘルプを表示

-V, --version：バージョンを表示
